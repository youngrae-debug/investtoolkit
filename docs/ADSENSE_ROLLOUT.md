# AdSense 운영 전 확인

## 현재 구현

- 가이드 상세 글의 두 번째 본문 뒤에 수동 반응형 광고 단위 한 개만 배치합니다.
- 광고 선택을 허용한 뒤에만 `adsbygoogle.js`를 불러옵니다.
- 광고 요청 전에 `requestNonPersonalizedAds=1`을 설정합니다.
- 계산기와 정책 혜택 페이지에는 광고 컴포넌트와 광고 선택 안내를 렌더링하지 않습니다.
- 실제 계정의 공개 게시자 ID로 사이트 확인 메타 태그와 `/ads.txt`를 항상 생성합니다.
- 광고 슬롯 ID가 없으면 광고 UI와 네트워크 요청은 계속 비활성 상태입니다.

## 실제 계정 연결

1. 2026-07-19에 AdSense 사이트 목록에 `invetk.com`을 추가했습니다.
2. 공개 게시자 ID `ca-pub-6841289838521074`를 소유권 확인 메타 태그와 `/ads.txt` 기본값으로 연결했습니다.
3. 이 변경을 배포한 뒤 `https://invetk.com/ads.txt`와 `google-adsense-account` 메타 태그를 확인합니다.
4. AdSense에서 `HTML <meta> 태그를 삽입했습니다.`를 선택하고 소유권 확인과 검토 요청을 진행합니다.
5. 사이트가 승인되면 반응형 디스플레이 광고 단위 한 개를 만들고 `NEXT_PUBLIC_ADSENSE_GUIDE_SLOT_ID`를 설정해 다시 배포합니다.
6. AdSense의 사이트 상태가 `준비됨`인지 확인한 뒤 제한된 가이드 경로부터 광고를 관찰합니다.

## 개인정보와 보안

- 개인 맞춤이 아닌 광고도 게재빈도와 통합 보고를 위해 쿠키를 사용할 수 있으므로 현재 구현은 명시적 허용 전 광고 스크립트를 지연합니다.
- EEA, 영국, 스위스 이용자에게 광고를 제공하기 전에는 Google이 인증한 CMP를 연결하고 해당 지역 동의 흐름을 별도로 검증해야 합니다.
- AdSense 도메인은 변경될 수 있어 광고가 활성화된 가이드 상세 경로만 HTTPS 하위 리소스를 허용합니다. 폭넓은 공개 운영 전에는 Vinext의 nonce 전달 지원을 확인한 뒤 `nonce`와 `strict-dynamic` 기반의 엄격한 CSP로 교체하는 것이 좋습니다.

## 출시 점검

로컬에서는 먼저 `npm run verify:ads`를 실행합니다. 이 명령은 비활성 상태와 테스트 ID 활성 상태를 모두 검증하며 Google에 실제 광고 요청을 보내지 않습니다.

- 동의 전 AdSense 네트워크 요청 0건
- 허용 후 광고 요청의 `npa=1` 확인
- 계산기와 정책 혜택에서 광고 스크립트·태그 0개
- 모바일에서 광고가 본문과 CTA를 가리지 않는지 확인
- 광고 차단기와 광고 미게재 상태에서도 본문 여백이 과도하지 않은지 확인
