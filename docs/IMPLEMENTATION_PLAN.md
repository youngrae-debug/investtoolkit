# 구현 계획

최종 업데이트: 2026-07-21

## 제품 우선순위

첫 방문자는 계산 예시 또는 첫 CTA에서 세 단계 입력을 시작해 목표 금액·날짜, 지금까지 모은 돈, 매달 모을 돈을 입력한다. 60초 안에 원금 기준 부족분과 실행안 3개, 이번 달 행동 계획을 확인한다. 재방문자는 첫 화면에서 저장 계획 요약과 이번 달 돈 점검을 바로 연다. 월 적립액을 모를 때만 월급과 지출 보조 흐름을 사용한다. 수익률, 조건 비교, 상세 그래프, 저장은 첫 결과 이후에만 제공한다.

## 구현 순서

1. 순수 함수 기반 월별 시뮬레이션 엔진과 한국어 포맷 함수
2. 1/3, 2/3, 3/3 단계형 목표·현재·월 적립 입력
3. 목표 날짜 기준 부족분과 월 적립·시작 자금·기간 조정 실행안 계산
4. 선택한 실행안으로 이번 달 행동 계획 생성
5. 선택적 수익률 가정과 조건 바꿔보기
6. 명시적 localStorage 저장, 이번 달 돈 점검, 데이터 백업
7. 공유, 상세 그래프·계산 내역, 계산 기준과 법적 문서
8. 가이드, 메타데이터, sitemap, robots, 접근성
9. 단위·E2E 테스트와 프로덕션 빌드

## 데이터 경계

모든 금융 계산은 브라우저에서 수행한다. 분석 이벤트에는 단계 번호와 상태 같은 비금융 속성만 허용한다. 저장된 계획만 목표 날짜와 선택한 실행안, 이번 달 저축액과 선택적으로 입력한 수입·총지출을 포함한 버전 7 스키마로 localStorage에 보관하며 버전 1~6 데이터는 자동 이전한다.

## 현재 구현 구조

- `components/money-gps/money-gps-app.tsx`: 전체 단계와 저장·공유 흐름 조율
- `components/money-gps/money-input.tsx`: 만 원 입력, 원 단위 확인, 범위 오류
- `components/money-gps/cashflow-helper.tsx`: 월급·지출 보조 흐름과 키보드 포커스 관리
- `components/money-gps/monthly-money-check.tsx`: 저축액 단일 입력과 수입·총지출 기반 월간 돈 점검
- `components/money-gps/monthly-savings-trend.tsx`: 최근 3개월 계획·실제 저축 흐름, 지난달 비교와 다음 달 보완 안내
- `components/money-gps/use-money-gps-analysis.ts`: 입력값에서 시뮬레이션과 목표일 분석 파생
- `lib/simulation/growth.ts`: 월 수익률, 월말 잔액, 성장계수의 공통 원천
- `lib/simulation/engine.ts`: 이벤트를 포함한 최대 1,200개월 월별 시뮬레이션
- `lib/simulation/goal-solver.ts`: 목표 날짜 부족분과 세 실행안 계산
- `lib/storage/plans.ts`: 버전 7 저장, 이전 버전 마이그레이션, 백업 검증

저장 데이터가 없는 브라우저에서도 백업을 불러올 수 있다. 이번 달 돈 점검은 결과 화면에 현재 표시된 실행안의 월 적립액을 사용하고, 해당 실행안 스냅샷을 함께 갱신한다. 수입보다 총지출이 크면 적자를 그대로 보여주고 실제 저축액은 0원으로 기록한다. 유효한 월별 기록은 날짜순으로 정렬해 최근 3개월의 계획·실제 합계와 흐름만 표시한다.
