# INVETK 제품 엔지니어링 루프

범용 SaaS 점검표를 Money GPS의 실제 사용자 흐름과 금융 데이터 원칙에 맞춘 실행 기준입니다. 한 번의 실행은 가장 큰 사용자 위험을 줄이는 하나의 검증 가능한 개선 묶음으로 끝냅니다.

## 제품 목적과 핵심 흐름

INVETK는 목표 금액과 날짜를 정한 사용자가 현재 자산과 월 적립액을 입력해 부족분을 이해하고, 세 가지 해결안 중 하나를 이번 달 행동으로 바꾸는 브라우저 기반 도구입니다. 광고 수익은 계산 경험을 방해하지 않는 가이드 상세에서만 허용합니다.

핵심 흐름은 다음 순서를 유지합니다.

1. 60초 안에 목표 금액·날짜, 현재 자산, 월 적립액을 입력한다.
2. 연 수익률 0% 원금 기준 부족분을 먼저 본다.
3. 월 적립 확대, 시작 자금 조합, 기간 조정의 세 가지 해결안을 비교한다.
4. 한 방법을 선택해 이번 달 행동 계획을 확인한다.
5. 명시적으로 저장한 뒤에만 현재 브라우저에 계획을 남긴다.
6. 재방문 시 실제 저축액을 기록하고 계획 대비 차이를 확인한다.

## 변경 불가 원칙

- 금융 입력값과 계산 결과를 서버, URL, 로그, 오류 수집, 분석 이벤트로 보내지 않는다.
- 저장은 사용자가 직접 요청한 뒤 버전이 있는 localStorage 스키마로만 수행한다.
- 첫 결과는 항상 연 수익률 0%이며, 수익률은 결과 이후의 선택적 가정으로만 제공한다.
- 특정 금융상품, 종목, 수익률을 추천하거나 결과를 보장하지 않는다.
- 계산기와 정책 혜택 화면에는 광고를 배치하지 않는다.
- 일반 화면에 내부 용어인 Scenario Lab, Check-in, JSON을 노출하지 않는다.
- 원격 푸시와 배포는 사용자가 해당 실행에서 명시적으로 요청한 경우에만 진행한다.

## 실행 순서

### 1. 이해

`README.md`, `package.json`, `AGENTS.md`, App Router 경로, 환경 변수 계약, 계산·포맷·저장 모듈, 공통 레이아웃과 테스트를 확인합니다. 현재 변경과 사용자 파일을 먼저 구분합니다.

### 2. 문제 탐색과 우선순위

| 등급 | INVETK 기준 | 예시 |
| --- | --- | --- |
| Critical | 돈 계산 오류, 데이터 외부 전송·손실, 비밀 노출, 운영 불능 | 부족분 부호 반전, 저장 마이그레이션 손상 |
| High | 핵심 계산 흐름 중단, 모바일·키보드 사용 불가, 색인 차단, 주요 보안 헤더 누락 | 다음 단계 진행 불가, 메뉴 포커스 고립 |
| Medium | 신뢰·전환·성능·유지보수에 반복 비용을 만드는 문제 | 날짜 메타데이터 불일치, 실패 피드백 누락 |
| Low | 기능을 막지 않는 시각·문구·코드 정리 | 작은 간격 불일치, 오래된 보조 문구 |

우선순위는 심각도뿐 아니라 영향받는 사용자 수, 재현 빈도, 수정 위험, 검증 가능성을 함께 봅니다. Critical과 High를 먼저 처리하고, 같은 원인에서 생긴 안전한 Medium·Low만 한 묶음에 포함합니다.

### 3. 구현

- 계산은 `lib/simulation/`의 순수 함수에 두고 React에서 재구현하지 않습니다.
- 금액·날짜·기간 표시는 `lib/format/`을 재사용합니다.
- 실패, 빈 상태, 로딩, 완료 상태는 다음 행동을 알 수 있는 한국어로 표시합니다.
- 기존 컴포넌트와 토큰을 우선 사용하고 단일 사용 추상화는 만들지 않습니다.
- 모바일 우선, 44px 터치 영역, 키보드 포커스, 충분한 대비를 유지합니다.

### 4. 자동 검증

다음 검사를 모두 통과해야 완료로 판단합니다.

```bash
npm run typecheck
npm run lint
npm test
npm run test:e2e
npm run verify:ads
npm run build
```

계산·저장 변경에는 경계값 단위 테스트를, 핵심 사용자 행동 변경에는 Playwright 회귀 테스트를 추가합니다. 콘솔 오류와 처리되지 않은 Promise 거부도 실패로 봅니다.

### 5. 브라우저 검증

| 화면 | 기준 크기 | 필수 확인 |
| --- | --- | --- |
| Desktop | 1440×900 | 정보 위계, 헤더·푸터, 계산 결과 비교 |
| Tablet | 820×1180 | 메뉴 전환, 카드 재배치, 터치 영역 |
| Mobile | 412×915 | 3단계 입력, 가로 넘침, 고정 UI 겹침 |

마우스 없이 메뉴, 3단계 입력, 모달 열기·닫기, 해결안 선택, 저장까지 이동합니다. Escape 시 포커스 복귀와 오류 발생 시 가장 가까운 설명·포커스를 확인합니다. 현재 제품은 밝은 테마만 지원하므로 운영체제 다크 모드에서도 브라우저 기본 컨트롤이 임의로 어두워지지 않는지 확인합니다.

### 6. 성능·SEO·접근성·보안·전환 검토

- 성능: 모바일 LCP 2.5초, CLS 0.1, INP 200ms를 운영 목표로 삼고 초기 화면에 필요 없는 코드와 제3자 스크립트를 늦춥니다. 동의 전 광고·분석 스크립트 요청은 허용하지 않습니다.
- SEO: 고유 title·description, canonical, Open Graph, Twitter Card, robots, sitemap, 구조화 데이터 날짜와 화면 날짜의 일치를 확인합니다.
- 접근성: 시맨틱 구조, 한 개의 명확한 h1, 레이블·설명 연결, 대비, 200% 확대, 키보드 순서와 보이는 포커스를 확인합니다.
- 보안: CSP·HSTS·nosniff·frame 차단, 의존성 취약점, 입력 상한, 백업 파일 스키마 검증을 확인합니다. 계정과 서버 저장이 없는 현재 구조에는 인증·CSRF·API rate limit을 억지로 추가하지 않습니다.
- 전환: 첫 CTA에서 결과까지의 단계 수, 단계별 이탈, 해결안 선택, 행동 계획 복사, 명시적 저장, 재방문 업데이트를 봅니다. 금액이나 결과값은 측정하지 않습니다.

### 7. 기록과 다음 실행

변경 파일, 사용자 효과, 자동·브라우저 검증 결과, 남은 위험, 다음 스프린트를 남깁니다. 점수는 근거가 있는 현재 상태 평가이며 임의의 성과 수치로 사용하지 않습니다. 실제 사용자 5명 검증 전에는 전환 개선을 확정된 성과로 표현하지 않습니다.

## 완료 기준

- 핵심 흐름과 금융 데이터 원칙을 훼손하지 않았습니다.
- 변경에 비례한 회귀 테스트가 있고 모든 필수 검사가 통과합니다.
- 데스크톱·태블릿·모바일에서 가로 넘침, 콘솔 오류, 막힌 키보드 흐름이 없습니다.
- 메타데이터·구조화 데이터·sitemap과 화면 정보가 일치합니다.
- 남은 문제와 배포 여부가 명확히 기록되어 있습니다.
