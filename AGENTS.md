# INVETK Money GPS 개발 가이드

## 구조

- `app/`: App Router 페이지, 메타데이터, 사이트 공통 레이아웃
- `components/`: 단계형 계산기와 재사용 UI
- `lib/simulation/`: UI와 분리된 순수 금융 계산 함수
- `lib/format/`: 한국어 금액·기간·연월 포맷
- `lib/storage/`: 버전이 있는 브라우저 저장·백업
- `tests/`: 단위 및 핵심 사용자 흐름 테스트
- `docs/`: 제품 결정, 검증 계획, 로드맵

## 명령

- 패키지 관리자: npm
- 개발: `npm run dev`
- 타입 검사: `npm run typecheck`
- 린트: `npm run lint`
- 단위 테스트: `npm test`
- E2E: `npm run test:e2e`
- 프로덕션 빌드: `npm run build`

## 구현 원칙

- 계산 로직은 React 컴포넌트 안에 작성하지 않는다.
- 첫 결과 전에는 목표 금액, 지금까지 모은 돈, 매달 모을 돈만 요구한다.
- 첫 결과는 연 수익률 0% 원금 기준이다.
- 금융 입력값은 서버, URL, 분석 이벤트로 보내지 않는다.
- 저장은 사용자의 명시적 행동 뒤 localStorage에만 수행한다.
- 일반 화면에는 `Scenario Lab`, `Check-in`, `JSON`을 노출하지 않는다.
- 한국어 시스템 글꼴, 모바일 우선, 키보드 접근성, 충분한 대비를 유지한다.
- 특정 금융상품 추천, 수익 보장, 자극적인 투자 표현을 사용하지 않는다.

## 완료 조건

- 3단계 첫 계산, 결과, 조건 변경, 저장, 이번 달 업데이트, 백업이 실제 동작한다.
- 계산 엔진 경계값과 핵심 사용자 흐름 테스트가 통과한다.
- 타입 검사, 린트, 프로덕션 빌드가 통과한다.
- 방법론·개인정보·약관·가이드·404·SEO 경로가 제공된다.
